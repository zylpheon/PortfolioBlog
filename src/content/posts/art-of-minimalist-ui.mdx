## Less, but better

Dieter Rams' famous principle — *weniger, aber besser* — less, but better — applies to UI design with almost uncomfortable precision. Every element on a screen either earns its place or costs the user attention. There's no neutral.

Minimalism in UI isn't about removing things until it looks "clean". It's about making deliberate decisions about what stays.

## The hierarchy problem

Most cluttered interfaces aren't cluttered because designers added too much. They're cluttered because **nothing was prioritised**.

When everything is emphasised, nothing is. When there are six call-to-action buttons at the same visual weight, the user has to figure out which one matters. That cognitive load is entirely avoidable.

<Callout type="tip">
  Before adding any UI element, ask: if this wasn't here, would the user be unable to complete their goal? If the answer is no, seriously consider removing it.
</Callout>

## Typography does the heavy lifting

In a minimal UI, typography carries enormous responsibility. The right typeface pairing can:

- Establish clear hierarchy without colour or icons
- Set the emotional tone of the product
- Guide the eye through the screen in the intended order

A useful constraint: **limit yourself to two typefaces**. One display font for headings and one body font for everything else. Within each, you have weight and size to create hierarchy — that's usually more than enough.

## Whitespace is not empty space

The most common mistake when attempting minimalism is treating whitespace as "wasted" space. The opposite is true.

Generous whitespace:
- Makes content easier to scan
- Creates perceived quality and intentionality
- Lets the most important elements breathe and assert themselves

Look at high-end print editorial design. The text-to-white ratio would horrify a digital product manager worried about "above the fold." And yet it feels effortless to read.

## Colour: commit to a constraint

Pick one primary colour. Use it sparingly — for the single most important interactive element on a given screen. Everything else should be greyscale or near-greyscale.

This is not a creative limitation. It's a forcing function that makes your primary actions unmistakeable and gives you precise control over where the eye lands.

## The hardest part: resisting additions

The pressure to add is constant. Stakeholders want features. Data suggests users sometimes don't find something. The instinct is to add a tooltip, a label, a button.

The better question is: why didn't they find it? Often the answer is that the navigation structure itself is unclear, or the content hierarchy is wrong. Adding more UI to compensate for a structural problem compounds the original issue.

---

Minimalism demands more decision-making upfront, not less. You're spending more time thinking carefully about every pixel rather than less. But the result — an interface that feels inevitable and effortless — is worth it.
